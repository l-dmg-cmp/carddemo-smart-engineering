Título do Documento: Data Integrity Guardian - Especificação do Repositório de Códigos de Pesquisa

Descrição Resumida: Este documento descreve a funcionalidade e as regras de negócios do módulo “Data Integrity Guardian”, com foco específico no repositório de códigos de pesquisa (CSLKPCDY.cpy). Este módulo desempenha um papel crucial na manutenção da precisão e consistência dos dados, definindo valores aceitáveis para códigos de área de telefone, códigos de estado dos EUA e combinações de código de estado dos EUA + primeiros dois dígitos do CEP. Isso garante que apenas dados válidos e padronizados entrem no sistema.

Histórias do Usuário: Como analista de dados, preciso ter certeza de que apenas códigos de área de telefone, códigos de estado dos EUA e códigos postais válidos sejam usados em nossos sistemas para manter a integridade dos dados e garantir uma análise precisa.

Épico Relacionado: 9 - Utilitários do Sistema

Requisitos Técnicos:

  • Validação de Código de Área de Telefone: Esta validação garante que qualquer número de telefone inserido no sistema esteja de acordo com o Plano de Numeração Norte-Americano estabelecido.
    • Entrada: Um código de área de telefone de três caracteres (XXX).
    • Cálculo: O sistema verifica se o código de área inserido está presente na lista VALID-PHONE-AREA-CODE dentro do arquivo CSLKPCDY.cpy.
    • Saída: Um valor booleano indicando se o código de área é válido (TRUE) ou não (FALSE).
  • Validação de Código de Estado dos EUA: Esta validação garante que todos os códigos de estado usados sejam as abreviações padrão de duas letras.
    • Entrada: Um código de estado dos EUA de dois caracteres (XX).
    • Cálculo: O sistema verifica se o código de estado inserido existe na lista VALID-US-STATE-CODE dentro do arquivo CSLKPCDY.cpy.
    • Saída: Um valor booleano indicando se o código de estado é válido (TRUE) ou não (FALSE).
  • Validação de Estado + CEP: Esta validação garante que o código de estado inserido e os primeiros dois dígitos do CEP sejam geograficamente compatíveis.
    • Entrada:
      • Um código de estado dos EUA de dois caracteres (XX).
      • Os primeiros dois dígitos do CEP (XX).
    • Cálculo: O sistema combina o código do estado e os primeiros dois dígitos do CEP (XXXX) e verifica se essa combinação está presente na lista VALID-US-STATE-ZIP-CD2-COMBO dentro do arquivo CSLKPCDY.cpy.
    • Saída: Um valor booleano indicando se a combinação de estado e CEP é válida (TRUE) ou não (FALSE).

Configurações:

  • CSLKPCDY.cpy: Este arquivo contém as seguintes listas:
    • VALID-PHONE-AREA-CODE: Lista de códigos de área de telefone norte-americanos válidos.
    • VALID-US-STATE-CODE: Lista de códigos de estado dos EUA válidos.
    • VALID-US-STATE-ZIP-CD2-COMBO: Lista de combinações válidas de códigos de estado dos EUA e os primeiros dois dígitos dos códigos postais.

Melhorias de Código:

  • Tratamento de Erros: Implemente códigos de erro e mensagens padronizados para cada regra de validação para fornecer feedback mais específico aos usuários.
  • Log: Registre as validações bem-sucedidas e as falhas de validação para fins de auditoria e solução de problemas.
  • Atualizações Dinâmicas: Explore maneiras de atualizar as listas de pesquisa dinamicamente (por exemplo, de uma fonte externa) para acomodar alterações em códigos de área, códigos de estado ou regulamentos de CEP.

Melhorias de Segurança:

  • Controle de Acesso: Implemente controles de acesso rígidos para evitar modificações não autorizadas no arquivo CSLKPCDY.cpy.
  • Verificações de Integridade: Implemente somas de verificação ou outras verificações de integridade para o arquivo CSLKPCDY.cpy para detectar quaisquer alterações não autorizadas.
