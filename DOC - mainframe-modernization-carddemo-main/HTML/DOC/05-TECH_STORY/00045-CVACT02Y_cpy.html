Título do Documento: Especificação da Estrutura de Dados do Cartão

Descrição Resumida: Este documento define a estrutura para armazenar informações de cartão de crédito no aplicativo CardDemo. Ele garante o armazenamento consistente e organizado de detalhes cruciais do cartão, como número do cartão, ID da conta, CVV, nome do titular do cartão, data de validade e status ativo.

Histórias do Usuário: Como desenvolvedor de sistema, preciso de uma maneira padronizada de representar dados de cartão de crédito no aplicativo CardDemo para garantir a consistência e integridade dos dados.

Epic Relacionado: 3 - Gestão de Cartão de Crédito

Requisitos Técnicos:

  • Definição da Estrutura de Dados: O código define uma estrutura de dados COBOL chamada CARD-RECORD para representar informações de cartão de crédito.
    • CARD-NUM: Um campo PIC X(16) para armazenar o número do cartão de crédito de 16 dígitos.
    • CARD-ACCT-ID: Um campo PIC 9(11) para armazenar o ID da conta de 11 dígitos associado ao cartão.
    • CARD-CVV-CD: Um campo PIC 9(03) para armazenar o Valor de Verificação do Cartão (CVV) de 3 dígitos.
    • CARD-EMBOSSED-NAME: Um campo PIC X(50) para armazenar o nome do titular do cartão como está em relevo no cartão.
    • CARD-EXPIRAION-DATE: Um campo PIC X(10) para armazenar a data de validade do cartão.
    • CARD-ACTIVE-STATUS: Um campo PIC X(01) para armazenar o status ativo do cartão.
    • FILLER: Um campo de preenchimento PIC X(59) para garantir um comprimento de registro consistente.
  • Validação de Dados:
    • O sistema deve validar o formato do número do cartão de crédito, a data de validade e o status ativo durante a entrada ou processamento.
    • O CVV deve ser tratado com segurança e não deve ser armazenado em texto simples.
  • Segurança de Dados:
    • Os dados do cartão de crédito devem ser criptografados em trânsito e em repouso.
    • O acesso aos dados do cartão de crédito deve ser restrito apenas ao pessoal autorizado.

Modelos Relacionados

  • Cartão: Este modelo representa um cartão de crédito e suas informações associadas.
    • cardNumber: String - O número do cartão de crédito de 16 dígitos.
    • accountId: Long - O identificador exclusivo da conta associada ao cartão.
    • cvv: String - O Valor de Verificação do Cartão (CVV) de 3 dígitos.
    • embossedName: String - O nome do titular do cartão como está em relevo no cartão.
    • expirationDate: Date - A data em que o cartão de crédito expira.
    • activeStatus: String - O status ativo do cartão (por exemplo, “ATIVO”, “INATIVO”).

Melhorias de Código:

  • Validação de Dados: Implemente regras robustas de validação de dados para cada campo na estrutura CARD-RECORD para evitar que dados inválidos sejam armazenados.
  • Tratamento de Erros: Inclua rotinas de tratamento de erros para lidar normalmente com situações como números de cartão inválidos, cartões expirados ou erros de banco de dados.
  • Melhorias de Segurança:
    • Criptografe os campos CARD-NUM e CARD-CVV-CD usando um algoritmo de criptografia forte.
    • Implemente mecanismos de controle de acesso para restringir o acesso aos dados do cartão de crédito.
  • Log: Implemente mecanismos de log para rastrear todas as operações realizadas nos dados do cartão de crédito para fins de auditoria.

Melhorias de Segurança:

  • Criptografia: Criptografe os campos CARD-NUM, CARD-CVV-CD e quaisquer outros campos confidenciais na estrutura CARD-RECORD usando um algoritmo de criptografia forte.
  • Controle de Acesso: Implemente medidas rígidas de controle de acesso para garantir que apenas pessoal autorizado possa acessar, modificar ou processar informações de cartão de crédito.
  • Armazenamento Seguro: Armazene as chaves de criptografia com segurança, separadas dos dados do cartão de crédito, para evitar descriptografia não autorizada.
