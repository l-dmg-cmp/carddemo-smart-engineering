Título do Documento: CardDemo Sistema de Gerenciamento de Usuários - Especificação do Programa Adicionar Novo Usuário

Descrição Resumida: Este documento descreve as especificações técnicas para o programa COBOL COUSR01C, responsável por adicionar novos usuários ao aplicativo CardDemo. Ele abrange a validação de dados, verificações de segurança, armazenamento de dados do usuário e interação com o CICS.

Histórias do Usuário: Como administrador do CardDemo, preciso poder adicionar novos usuários ao sistema para que eles possam acessar o aplicativo e executar suas funções atribuídas.

Épico Relacionado: 6 - Gerenciamento de Usuários e Segurança

Requisitos Técnicos:

  • Validação de Entrada do Usuário:
    • O programa valida as entradas do usuário da estrutura de dados COUSR1AI (recebida da tela “Adicionar Usuário”) para os seguintes campos:
      • Nome (FNAMEI): Não pode estar vazio.
      • Sobrenome (LNAMEI): Não pode estar vazio.
      • ID do Usuário (USERIDI): Não pode estar vazio e deve ser exclusivo (verificado no arquivo USRSEC).
      • Senha (PASSWDI): Não pode estar vazia.
      • Tipo de Usuário (USRTYPEI): Não pode estar vazio e deve ser “Regular” ou “Admin”.
  • Armazenamento de Dados:
    • O programa grava os dados do novo usuário no arquivo VSAM USRSEC.
    • A estrutura de dados SEC-USER-DATA é usada para formatar os dados antes de gravá-los no arquivo.
    • O campo SEC-USR-ID é usado como chave para o arquivo USRSEC.
  • Tratamento de Erros:
    • Se ocorrer um erro durante a validação da entrada:
      • O sinalizador WS-ERR-FLG é definido como ‘Y’.
      • Uma mensagem de erro apropriada é movida para WS-MESSAGE.
      • O cursor é definido para o campo com erro na tela COUSR1AI.
      • A rotina SEND-USRADD-SCREEN é executada para exibir novamente a tela com a mensagem de erro.
    • Se ocorrer um erro durante a operação de gravação no arquivo USRSEC:
      • Uma mensagem de erro é exibida ao usuário.
      • O programa não prossegue com a adição do usuário.
  • Interação CICS:
    • O programa usa os seguintes comandos CICS:
      • RECEIVE: Para receber entrada do usuário do mapa COUSR1A.
      • SEND: Para enviar o mapa COUSR1A ao terminal do usuário.
      • WRITE: Para gravar o novo registro do usuário no arquivo USRSEC.
      • XCTL: Para transferir o controle para outro programa.
  • Manuseio de Tela:
    • O programa interage com o mapa COUSR1A e o conjunto de mapas COUSR01 para entrada e exibição do usuário.
    • A rotina SEND-USRADD-SCREEN preenche e envia a tela ao usuário.
    • A rotina RECEIVE-USRADD-SCREEN recebe a entrada do usuário da tela.
  • Data e Hora:
    • O programa recupera a data e hora atuais usando a função CURRENT-DATE e as formata para exibição na tela.
  • Segurança:
    • O programa não lida diretamente com criptografia ou armazenamento de senha. Supõe-se que o arquivo USRSEC ou o sistema subjacente lida com a segurança da senha.
  • Regras de Negócios:
    • Campos Obrigatórios: Todos os detalhes do usuário (Nome, Sobrenome, ID do Usuário, Senha, Tipo de Usuário) são obrigatórios.
    • ID de Usuário Exclusivo: O sistema impõe a exclusividade dos IDs de Usuário para evitar conflitos e garantir que cada usuário possa ser identificado corretamente.

Modelos Relacionados:

  • SEC-USER-DATA:
    • SEC-USR-ID X(XX): Identificador exclusivo do usuário.
    • SEC-USR-FNAME X(XX): Nome do usuário.
    • SEC-USR-LNAME X(XX): Sobrenome do usuário.
    • SEC-USR-PWD X(XX): Senha do usuário.
    • SEC-USR-TYPE X(XX): Função do usuário (Regular ou Admin).

Configurações:

  • WS-USRSEC-FILE: Nome do arquivo para o banco de dados de usuários (USRSEC).

Melhorias de Código:

  • Segurança de Senha: Implemente o hash seguro de senha antes de armazená-la no arquivo USRSEC.
  • Registro de Erros: Registre erros em um arquivo ou banco de dados para fins de depuração e auditoria.
  • Limpeza de Entrada: Limpe as entradas do usuário para evitar vulnerabilidades como injeção de SQL (embora menos crítica em um ambiente COBOL/VSAM, é uma boa prática).
  • Modularização: Divida parágrafos grandes em funções menores e mais gerenciáveis para melhor organização do código e legibilidade.
  • Comentários de Código: Adicione comentários mais detalhados para explicar a lógica por trás de seções específicas do código.

Melhorias de Segurança:

  • Complexidade da Senha: Imponha regras de complexidade de senha (comprimento mínimo, caracteres especiais, etc.).
  • Controle de Acesso: Implemente um controle de acesso mais rígido ao programa COUSR01C, garantindo que apenas pessoal autorizado possa adicionar novos usuários.
  • Auditoria: Registre todos os eventos de criação de usuário (incluindo quem adicionou o usuário, carimbo de data/hora, etc.) para fins de auditoria.
