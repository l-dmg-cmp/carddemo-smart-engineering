Título do Documento: Validação e Processamento de Dados de Cartão

Descrição Resumida: Este programa valida dados de cartão de crédito de um arquivo chamado carddata.txt. Ele verifica o formato do número do cartão usando o Algoritmo de Luhn, o comprimento do número do cartão e a validade da data de expiração. Os registros válidos são gravados em um arquivo de saída e os registros inválidos são registrados.

Histórias do Usuário: Como analista de dados, desejo garantir que apenas registros de cartão de crédito válidos sejam usados para análise, portanto, preciso de um programa para validar os dados e fornecer um arquivo de saída apenas com os registros válidos.

Épico Relacionado: 3 - Gestão de Cartão de Crédito

Requisitos Técnicos:

  • Extração de Dados:

    • Entrada: Arquivo carddata.txt.
    • Processo: Leia o arquivo de entrada linha por linha. Para cada linha:
      • Divida a linha em campos com base em espaços.
      • Extraia card_number, customer_name e expiration_date dos campos.
    • Saída: Uma lista de registros, cada um contendo card_number, customer_name e expiration_date.

  • Validação do Número do Cartão:

    • Entrada: card_number (string).
    • Processo: Implemente o Algoritmo de Luhn:
      1. Começando do dígito mais à direita (excluindo o dígito de verificação, se houver), dobre cada segundo dígito. Se a duplicação resultar em um número de dois dígitos, subtraia 9 dele.
      2. Some todos os dígitos (incluindo os dígitos não modificados e os resultados da etapa 1).
      3. Se a soma for um múltiplo de 10, o número do cartão é válido.
    • Saída: True se válido, False caso contrário.

  • Validação do Comprimento do Número do Cartão:

    • Entrada: card_number (string).
    • Processo: Verifique se o comprimento de card_number está dentro do intervalo aceitável (13-19 dígitos).
    • Saída: True se válido, False caso contrário.

  • Validação da Data de Expiração:

    • Entrada: expiration_date (string).
    • Processo:
      1. Verifique se expiration_date corresponde ao formato YYYY-MM-DD.
      2. Converta expiration_date em um objeto de data.
      3. Compare a data analisada com a data atual.
    • Saída: True se válido, False caso contrário.

  • Geração de Saída:

    • Entrada: Lista de registros validados.
    • Processo: Grave os registros validados em um arquivo de saída.
    • Saída: Arquivo de saída contendo registros de cartão de crédito válidos.

Modelos Relacionados:

  • CreditCard:
    • card_number: String (13-19 dígitos).
    • customer_name: String.
    • expiration_date: Data (YYYY-MM-DD).

Configurações:

  • input_file_path: carddata.txt
  • output_file_path: validated_card_data.txt
  • log_file_path: card_validation_errors.log
  • date_format: YYYY-MM-DD

Melhorias de Código:

  • Implementar o registro em log para melhor rastreamento e depuração.
  • Adicionar tratamento de erros para operações de arquivo (por exemplo, arquivo não encontrado, erros de leitura / gravação).
  • Considere o uso de uma biblioteca de validação de dados para regras de validação mais robustas e relatório de erros.
  • Otimizar o desempenho ao lidar com arquivos muito grandes (por exemplo, leitura e processamento em partes).

Melhorias de Segurança:

  • Armazene dados confidenciais (números de cartão de crédito) com segurança, seguindo os padrões PCI DSS.
  • Implementar controle de acesso para restringir o acesso não autorizado ao programa, arquivos de dados e logs.
  • Usar criptografia para dados em repouso e em trânsito.
  • Auditar regularmente o programa e os procedimentos de tratamento de dados para vulnerabilidades de segurança.
