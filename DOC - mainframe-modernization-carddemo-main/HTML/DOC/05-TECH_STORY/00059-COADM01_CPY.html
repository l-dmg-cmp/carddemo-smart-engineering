Título do Documento: COADM01 - Estrutura de Dados Administrativos

Descrição Resumida: Este documento descreve o copybook COBOL COADM01.CPY, que define estruturas de dados para lidar com informações administrativas, incluindo títulos de tela, nomes de programas, opções de menu e mensagens de erro. Ele enfatiza a representação dupla de dados para processamento interno eficiente e exibição amigável ao usuário.

Histórias do Usuário: Como desenvolvedor, preciso de uma maneira padronizada de gerenciar e exibir informações administrativas em diferentes telas e módulos do aplicativo CardDemo. Isso garante uma experiência consistente ao usuário e simplifica o desenvolvimento, fornecendo estruturas de dados reutilizáveis.

Épico Relacionado: 9 - Utilitários do Sistema

Requisitos Técnicos:

  • Estruturas de Dados:

    • COADM1AI: Define campos de dados administrativos em um formato numérico compacto (COMP) para processamento interno eficiente dentro do programa COBOL.
    • COADM1AO: Redefine COADM1AI para representar os mesmos dados em um formato baseado em caracteres adequado para exibição em telas ou relatórios. Esta estrutura usa campos de caracteres (PIC X) para legibilidade do usuário.

  • Mapeamento de Campo:

    • Cada campo em COADM1AI possui um campo correspondente em COADM1AO com a mesma finalidade, mas um tipo de dado diferente. Por exemplo, TRNNAMEL (numérico) mapeia para TRNNAMEO (alfanumérico).
    • O mapeamento entre as estruturas garante que os dados possam ser facilmente transferidos entre o processamento interno e a lógica de exibição.

  • Tipos e Tamanhos de Dados:

    • Campos numéricos (COMP) em COADM1AI são provavelmente definidos com PIC S9(n) para armazenar números com sinal de forma eficiente.
    • Campos alfanuméricos (PIC X(n)) em COADM1AO são usados para armazenar dados de caracteres, como títulos de tela e opções de menu.

  • Atributos Visuais (Possível Integração BMS):

    • Campos em COADM1AO terminando com ‘C’, ‘P’, ‘H’, ‘V’, ‘O’ (por exemplo, TRNNAMEC, TITLE01P) sugerem integração com o Basic Mapping Support (BMS) para controlar atributos visuais como cor, destaque e posicionamento na tela.

Melhorias de Código:

  • Documentação: Adicione comentários para esclarecer o propósito de cada campo e a estrutura geral.
  • Convenções de Nomenclatura: Use nomes mais descritivos para os campos (por exemplo, TRANSACTION-NAME em vez de TRNNAMEL) para melhorar a legibilidade.
  • Validação de Dados: Considere adicionar rotinas de validação de dados para garantir a integridade dos dados administrativos, especialmente se eles estiverem sendo preenchidos a partir de fontes externas.

Melhorias de Segurança:

  • Controle de Acesso: Se os dados administrativos incluem informações confidenciais (por exemplo, IDs de usuário, nomes de programas), implemente mecanismos de controle de acesso para restringir a visualização ou modificação não autorizada.
  • Trilhas de Auditoria: Registre quaisquer alterações feitas nos dados administrativos para fins de auditoria e rastreamento.
