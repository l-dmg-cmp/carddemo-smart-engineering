Documento de Título: Especificação de Registro de Conta CardDemo

Descrição Resumida:

Este documento detalha a estrutura do ACCOUNT-RECORD usado no sistema CardDemo. Este registro define o layout para armazenar informações sobre a conta de cartão de crédito de cada cliente, abrangendo detalhes como ID da conta, status, saldo, limites de crédito, datas e afiliação ao grupo.

Histórias de Usuário:

Como administrador do sistema de cartão de crédito, preciso de uma maneira padronizada de armazenar e acessar informações detalhadas sobre a conta de cada cliente para garantir o processamento preciso de transações, gerenciamento de saldo e relatórios.

Épico Relacionado:

2 - Gestão de Contas

Requisitos Técnicos:

Este código define uma estrutura de dados em COBOL para representar a conta de um cliente.

  • Estrutura ACCOUNT-RECORD: Esta estrutura agrupa todos os dados relacionados à conta.
    • ACCT-ID: Um campo numérico exclusivo de 11 dígitos que representa o identificador da conta.
    • ACCT-ACTIVE-STATUS: Um campo de 1 caractere (‘Y’ ou ‘N’) indicando se a conta está ativa.
    • ACCT-CURR-BAL: Um campo numérico com sinal com duas casas decimais, armazenando o saldo atual da conta.
    • ACCT-CREDIT-LIMIT: Um campo numérico com sinal com duas casas decimais, indicando o crédito máximo permitido.
    • ACCT-CASH-CREDIT-LIMIT: Um campo numérico com sinal com duas casas decimais, especificando o limite máximo de saque em dinheiro.
    • ACCT-OPEN-DATE, ACCT-EXPIRAION-DATE, ACCT-REISSUE-DATE: Campos de data no formato YYYY-MM-DD armazenando as datas de abertura, expiração e última reemissão da conta.
    • ACCT-CURR-CYC-CREDIT, ACCT-CURR-CYC-DEBIT: Campos numéricos com sinal com duas casas decimais, rastreando o total de créditos e débitos no ciclo atual.
    • ACCT-ADDR-ZIP: Um campo alfanumérico de 10 caracteres armazenando o CEP associado à conta.
    • ACCT-GROUP-ID: Um campo alfanumérico de 10 caracteres indicando a afiliação do grupo da conta (por exemplo, ‘PREMIUM’).
    • FILLER: Um campo alfanumérico de 178 caracteres reservado para uso futuro.

Modelos Relacionados

  • Conta: Representa a conta de cartão de crédito de um cliente.
    • accountID: String - Identificador exclusivo para a conta.
    • activeStatus: Boolean - Indica se a conta está ativa ou não.
    • currentBalance: Decimal - O saldo atual da conta.
    • creditLimit: Decimal - O crédito máximo permitido para a conta.
    • cashCreditLimit: Decimal - A quantidade máxima de dinheiro que pode ser retirada da conta.
    • openDate: Date - A data em que a conta foi aberta.
    • expirationDate: Date - A data de expiração da conta.
    • reissueDate: Date - A data da última reemissão da conta.
    • currentCycleCredit: Decimal - O total de créditos aplicados à conta no ciclo atual.
    • currentCycleDebit: Decimal - O total de débitos aplicados à conta no ciclo atual.
    • addressZip: String - O CEP associado à conta.
    • groupID: String - O grupo ao qual a conta pertence.

Melhorias de Código:

  • Validação de Dados: Implemente verificações de validação de dados para todos os campos dentro da estrutura ACCOUNT-RECORD para garantir a integridade dos dados. Por exemplo:
    • Valide o campo ACCT-ACTIVE-STATUS para garantir que ele contenha apenas ‘Y’ ou ‘N’.
    • Verifique se as datas estão no formato correto e são logicamente válidas (por exemplo, ACCT-EXPIRAION-DATE deve ser posterior a ACCT-OPEN-DATE).
    • Verifique se há valores negativos nos campos de saldo e limite de crédito, se não for logicamente permitido.
  • Tratamento de Erros: Incorpore mecanismos de tratamento de erros para gerenciar dados inválidos ou situações inesperadas durante o processamento de dados.
  • Documentação: Adicione comentários abrangentes ao código para explicar a finalidade de cada campo, tipos de dados usados e quaisquer regras de validação aplicadas. Isso melhorará a manutenção e o entendimento do código.

Melhorias de Segurança:

  • Criptografia de Dados: Considere criptografar dados confidenciais dentro do ACCOUNT-RECORD, como números de contas, saldos e limites de crédito, durante o armazenamento e a transmissão.
  • Controle de Acesso: Implemente medidas rígidas de controle de acesso para restringir o acesso não autorizado às informações da conta. Somente pessoal e aplicativos autorizados devem ter permissões para ler, modificar ou excluir dados da conta.
  • Log de Auditoria: Mantenha um log de auditoria de todos os acessos e modificações feitos aos dados do ACCOUNT-RECORD. Este log deve registrar o usuário, a data e hora e a natureza da operação realizada. Isso ajuda a rastrear violações de dados e garante a responsabilidade.
