Documento de Título: CardDemo - Especificação da Estrutura do Grupo de Divulgações

Descrição Resumida: Este documento descreve a estrutura e função do Disclosure Group dentro do aplicativo CardDemo. Esta estrutura é crucial para determinar a taxa de juros aplicada a transações específicas de cartão de crédito com base em fatores como Account Type, Transaction Type e Transaction Category.

Histórias de Usuário: Como Administrador do Sistema de Cartão de Crédito, preciso definir diferentes taxas de juros para vários tipos e categorias de transações para que possamos aplicar encargos de juros apropriados às contas dos clientes.

Epic Relacionado: 3 - Gestão de Cartão de Crédito

Requisitos Técnicos:

  • Estrutura de Dados:
    • O sistema deve fornecer uma estrutura de dados para armazenar informações do Disclosure Group, incluindo:
      • Account Group ID: Um código que identifica o grupo da conta (por exemplo, Gold, Platinum).
      • Transaction Type Code: Um código que representa o tipo de transação (por exemplo, Compra, Adiantamento em Dinheiro).
      • Transaction Category Code: Um código para a categoria de transação (por exemplo, Mercearias, Viagens).
      • Interest Rate: O valor percentual da taxa de juros aplicada a este grupo.
  • Validação de Dados:
    • O sistema deve garantir que cada Disclosure Group tenha uma combinação única de Account Group ID, Transaction Type Code e Transaction Category Code.
    • O valor da Interest Rate deve estar dentro de um intervalo válido (por exemplo, 0,00% a 99,99%).
  • Recuperação de Dados:
    • O sistema deve ser capaz de recuperar com eficiência a taxa de juros correta para uma determinada transação com base em seu Account Group ID, Transaction Type Code e Transaction Category Code.
  • Manutenção de Dados:
    • O sistema deve permitir que usuários autorizados:
      • Adicionem novos Disclosure Groups.
      • Modifiquem Disclosure Groups existentes.
      • Desativem Disclosure Groups desatualizados.

Modelos Relacionados:

  • Disclosure Group: Representa uma combinação de conta, tipo de transação e categoria com uma taxa de juros específica.
    • DIS-ACCT-GROUP-ID String: Identificador exclusivo para o grupo da conta.
    • DIS-TRAN-TYPE-CD String: Identificador exclusivo para o tipo de transação.
    • DIS-TRAN-CAT-CD String: Identificador exclusivo para a categoria de transação.
    • DIS-INT-RATE Decimal: Taxa de juros aplicada às transações pertencentes a este grupo.

Configurações:

  • Maximum Interest Rate: 99.99

Melhorias de Código:

  • Tratamento de Erros: Implemente um tratamento de erros robusto para processos de validação e recuperação de dados, incluindo erros de registro e notificação de administradores.
  • Otimização de Desempenho: Use mecanismos de indexação ou cache apropriados para otimizar o desempenho da recuperação de dados, especialmente ao lidar com um grande número de Disclosure Groups.
  • Modularidade de Código: Projete o código com a modularidade em mente, separando as definições da estrutura de dados, lógica de validação, funções de recuperação e operações de manutenção em módulos distintos para melhor manutenção.

Melhorias de Segurança:

  • Controle de Acesso: Implemente medidas rigorosas de controle de acesso para garantir que apenas usuários autorizados possam modificar os dados do Disclosure Group.
  • Registro de Auditoria: Mantenha registros de auditoria de todas as alterações feitas nos Disclosure Groups, incluindo quem fez a alteração, o que foi alterado e quando foi alterado.
