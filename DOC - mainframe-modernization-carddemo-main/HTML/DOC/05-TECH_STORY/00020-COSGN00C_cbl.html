Título do Documento: Aplicativo CardDemo - Especificação do Módulo de Iniciação de Sessão

Descrição Resumida: Este documento descreve as especificações para o módulo de Iniciação de Sessão (COSGN00C) dentro do aplicativo CardDemo, um sistema baseado em COBOL projetado para gerenciamento de cartão de crédito. O módulo garante autenticação segura do usuário, validando as credenciais do usuário em relação aos dados armazenados. Após a verificação bem-sucedida, ele concede aos usuários acesso ao aplicativo com base em suas funções definidas, direcionando-os para as funcionalidades apropriadas.

Histórias do Usuário: Como Representante de Atendimento ao Cliente, preciso fazer login no aplicativo CardDemo com segurança para poder acessar informações do cliente e realizar transações.

Épico Relacionado: 6 - Gestão de Usuários e Segurança

Requisitos Técnicos:

  • Exibir Tela de Iniciação de Sessão:

    • O módulo apresenta uma tela solicitando ao usuário que insira seu ID de Usuário e Senha.
    • Usa o copybook COSGN00 para a estrutura de dados da tela.
    • Usa a função SEND-SIGNON-SCREEN para enviar a tela para o terminal do usuário.

  • Capturar Entrada do Usuário:

    • O sistema captura o ID do Usuário e a Senha inseridos pelo usuário.
    • Usa o comando CICS RECEIVE para capturar dados da tela.

  • Validar Credenciais do Usuário:

    • Verifica se o ID do Usuário inserido existe no arquivo USRSEC.
    • Se encontrado, compara a Senha inserida com a Senha armazenada para aquele ID de Usuário.
    • Usa a função READ-USER-SEC-FILE para ler dados do usuário do arquivo USRSEC.
    • Emprega técnicas de validação de dados para verificar se há caracteres vazios ou inválidos no ID do Usuário e na Senha.

  • Determinar o Tipo de Usuário:

    • Após a validação bem-sucedida, o sistema identifica a função do usuário (por exemplo, Administrador ou Usuário Regular) a partir dos dados do usuário.
    • O tipo de usuário é armazenado no campo SEC-USR-TYPE no arquivo USRSEC.

  • Direcionar o Usuário para o Menu Apropriado:

    • Com base na função do usuário, o módulo redireciona o usuário para a parte relevante do aplicativo (por exemplo, menu do Administrador ou Menu Principal).
    • Usa o comando CICS XCTL para transferir o controle para outros programas (COADM01C para Administrador, COMEN01C para usuários regulares).

Modelos Relacionados:

  • SEC-USER-DATA: Representa o registro do usuário recuperado do arquivo USRSEC.

    • SEC-USR-PWD: Armazena a Senha criptografada para o usuário.
    • SEC-USR-TYPE: Armazena o tipo de usuário (por exemplo, ‘A’ para Administrador, ‘U’ para Usuário Regular).

  • CARDDEMO-COMMAREA: Área de comunicação usada para passar dados entre programas.

    • CDEMO-USER-ID: Armazena o ID do Usuário para uso por outros módulos.
    • CDEMO-USER-TYPE: Armazena o tipo de usuário para uso por outros módulos.

Configurações:

  • WS-USRSEC-FILE: Nome do arquivo para o arquivo de segurança do usuário. O valor padrão é USRSEC.
  • WS-TRANID: ID da transação para o módulo de iniciação de sessão. O valor padrão é CC00.

Melhorias de Código:

  • Segurança de Senha: Implementar um mecanismo de armazenamento de Senha mais robusto, como hashing unidirecional, em vez de armazenar Senhas em texto simples.
  • Tratamento de Erros: Fornecer mensagens de erro mais específicas para o usuário, como “ID de Usuário Inválido” ou “Senha Incorreta.”
  • Gerenciamento de Sessão: Implementar um mecanismo para rastrear sessões de usuário e impedir o acesso não autorizado.

Melhorias de Segurança:

  • Complexidade da Senha: Impor políticas de Senha forte, exigindo que os usuários definam Senhas complexas.
  • Autenticação Multifator (MFA): Considere implementar MFA para uma camada adicional de segurança.
  • Registro de Auditoria: Registrar todas as tentativas de iniciação de sessão, bem-sucedidas e malsucedidas, para auditoria e monitoramento de segurança.
