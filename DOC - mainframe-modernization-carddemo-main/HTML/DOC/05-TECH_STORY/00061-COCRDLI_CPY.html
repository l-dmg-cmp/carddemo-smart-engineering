Título do Documento: Especificação de Layout de Dados de Cartão de Crédito

Descrição Resumida: Este documento define a estrutura das informações de cartão de crédito usadas no sistema. Inclui dois layouts: CCRDLIAI para representação interna e CCRDLIAO para comunicação externa. Esses layouts garantem o tratamento consistente e seguro de dados confidenciais do cartão de crédito entre diferentes componentes do sistema e durante interações com entidades externas.

Histórias do Usuário: Como administrador do sistema, preciso de um formato padronizado para informações de cartão de crédito para que diferentes partes do sistema possam processar e trocar dados sem problemas.

Épico Relacionado: 3 - Gerenciamento de Cartão de Crédito

Requisitos Técnicos:

  • Definição da Estrutura de Dados:
    • Definir duas estruturas de dados COBOL: CCRDLIAI e CCRDLIAO.
    • CCRDLIAI: Representa o layout interno das informações do cartão de crédito.
    • CCRDLIAO: Representa o layout externo das informações do cartão de crédito.
  • Mapeamento de Campo:
    • Estabelecer um mapeamento claro entre os campos correspondentes em CCRDLIAI e CCRDLIAO.
    • Garantir a consistência e precisão dos dados durante as transformações de dados internos para externos e externos para internos.
  • Validação de Dados:
    • Implementar regras de validação de dados para cada campo em ambos os layouts.
    • Validação de tipo de dados: Impor tipos de dados corretos (por exemplo, alfanumérico, numérico).
    • Validação de comprimento do campo: Restringir os comprimentos dos campos aos limites definidos.
  • Considerações de Segurança:
    • Criptografar informações confidenciais do cartão de crédito (por exemplo, número do cartão) em ambos os layouts durante o armazenamento e transmissão.
    • Implementar mecanismos de controle de acesso para restringir o acesso não autorizado a COCRDLI.CPY.

Modelos Relacionados:

  • CCRDLIAI:
    • TRNNAME: String, Nome ou identificador da transação.
    • TITLE01: String, Título descritivo ou cabeçalho para informações do cartão de crédito.
    • CURDATE: Date, Data atual.
    • PGMNAME: String, Nome do programa.
    • TITLE02: String, Segundo título descritivo ou cabeçalho.
    • CURTIME: Time, Hora atual.
    • PAGENO: Integer, Número da página.
    • ACCTSID: String, Identificador da conta.
    • CARDSID: String, Identificador do cartão.
    • CRDSEL[1-7]: String, Sinalizadores ou indicadores relacionados à seleção do cartão de crédito.
    • CRDSTP[1-7]: String, Sinalizadores ou indicadores relacionados ao status ou tipo do cartão de crédito.
    • ACCTNO[1-7]: String, Números de conta.
    • CRDNUM[1-7]: String, Números do cartão de crédito.
    • CRDSTS[1-7]: String, Status do cartão de crédito.
    • INFOMSG: String, Mensagem informativa.
    • ERRMSG: String, Mensagem de erro.
  • CCRDLIAO:
    • (Mesmos campos que CCRDLIAI com representações diferentes para comunicação externa)

Configurações:

  • Nenhuma

Melhorias de Código:

  • Tratamento de Erros: Implementar mecanismos robustos de tratamento de erros para processos de validação e transformação de dados.
  • Documentação: Fornecer documentação abrangente para os layouts CCRDLIAI e CCRDLIAO, incluindo descrições de campo, tipos de dados e regras de validação.

Melhorias de Segurança:

  • Armazenamento Seguro: Armazene COCRDLI.CPY com segurança com controles de acesso apropriados para evitar modificações não autorizadas.
  • Revisão de Código: Conduzir revisões regulares de código de segurança para identificar e resolver potenciais vulnerabilidades.
