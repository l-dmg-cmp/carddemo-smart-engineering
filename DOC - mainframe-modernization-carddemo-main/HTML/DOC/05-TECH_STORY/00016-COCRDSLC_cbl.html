Título do Documento: Especificação do Programa de Recuperação de Detalhes de Cartão de Crédito

Descrição Resumida: Este documento detalha as especificações funcionais para um programa COBOL projetado para recuperar e exibir detalhes do cartão de crédito com base nos números de conta ou cartão fornecidos pelo usuário. Ele utiliza o CICS para processamento de transações e interage com um banco de dados VSAM chamado ‘CARDDAT’.

Histórias do Usuário: Como representante de atendimento ao cliente, preciso acessar rapidamente os detalhes do cartão de crédito de um cliente usando sua conta ou número do cartão para que eu possa auxiliá-lo com eficiência.

Epic Relacionado: 3 - Gestão de Cartão de Crédito

Requisitos Técnicos:

  • Validação de Entrada:
    • Validar se a entrada CC-ACCT-ID é numérica e tem 11 dígitos.
    • Validar se a entrada CC-CARD-NUM é numérica e tem 16 dígitos.
  • Interação com o Banco de Dados:
    • O programa lê dados de um arquivo VSAM chamado CARDDAT.
    • O programa usa o comando READ com a opção RIDFLD para pesquisar registros específicos com base em WS-CARD-RID-CARDNUM (número do cartão).
  • Recuperação de Dados:
    • Se um registro de cartão correspondente for encontrado, o programa recupera os detalhes do cartão para a estrutura de dados CARD-RECORD.
  • Tratamento de Erros:
    • Se ocorrer um erro durante o processamento do arquivo, o programa preenche o WS-FILE-ERROR-MESSAGE com detalhes como a operação (ERROR-OPNAME), nome do arquivo (ERROR-FILE) e códigos de resposta CICS (ERROR-RESP e ERROR-RESP2).
  • Interação com a Tela:
    • O programa interage com uma interface de usuário (provavelmente um terminal 3270) para receber entrada e exibir os detalhes do cartão recuperado.
    • O layout da tela e o mapeamento de dados são definidos nos copybooks COCRDSL e CCRDSLA.

Modelos Relacionados

  • CARD-RECORD: Representa um registro de cartão de crédito.
    • CARD-EMBOSSED-NAME: String - O nome em relevo no cartão de crédito.
    • CARD-EXPIRAION-DATE: String - A data de validade do cartão de crédito.
    • CARD-ACTIVE-STATUS: String - O status do cartão de crédito (por exemplo, ativo, inativo).

Configurações:

  • LIT-CARDFILENAME: CARDDAT - O nome do arquivo de banco de dados do cartão de crédito.

Melhorias de Código:

  • Implementar um mecanismo de tratamento de erros mais robusto:
    • Em vez de apenas mover mensagens de erro para WS-RETURN-MSG, considere usar uma abordagem estruturada de tratamento de erros.
    • Registrar erros em um arquivo ou banco de dados para auditoria e solução de problemas.
  • Melhorar a documentação do código:
    • Adicionar mais comentários para explicar a lógica e o propósito de diferentes seções de código.
    • Criar um documento separado com uma visão geral de alto nível da funcionalidade e arquitetura do programa.
  • Otimizar para desempenho:
    • Se o arquivo CARDDAT for grande, considere implementar indexação ou outras técnicas de aprimoramento de desempenho para recuperação de dados mais rápida.

Melhorias de Segurança:

  • Mascarar ou criptografar dados confidenciais do cartão:
    • Não armazenar ou exibir o número completo do cartão de crédito, código CVV ou outras informações confidenciais em texto não criptografado.
    • Implementar criptografia ou tokenização para dados confidenciais em repouso e em trânsito.
  • Implementar controle de acesso:
    • Restringir o acesso ao programa e ao arquivo CARDDAT apenas ao pessoal autorizado.
  • Registrar todas as tentativas de acesso a dados:
    • Registrar IDs de usuário, timestamps e ações executadas para fins de auditoria.
